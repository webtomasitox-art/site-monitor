# מנטר אתר אוטומטי בענן — מדריך הקמה

מערכת שבודקת את האתר אוטומטית כל כמה שעות (גם כשהמחשב כבוי),
ומציגה את השינויים שזוהו בדשבורד אינטרנטי. הכול חינם, דרך GitHub.

## מה יש כאן
- `monitor.js` — המנוע שמצלם ומשווה.
- `package.json` — רשימת הרכיבים.
- `.github/workflows/monitor.yml` — מפעיל את המנוע אוטומטית בענן.
- `docs/` — הדשבורד שמוצג בדפדפן (index.html) וקבצי הנתונים.

## הקמה (פעם אחת)
1. פתח חשבון חינמי ב-https://github.com
2. צור פרויקט (repository) חדש.
3. העלה את כל הקבצים מהתיקייה הזו, תוך שמירה על מבנה התיקיות.
4. בהגדרות הפרויקט: Settings → Actions → General → תחת
   "Workflow permissions" בחר **Read and write permissions** ושמור.
5. בהגדרות הפרויקט: Settings → Pages → תחת "Source" בחר את הענף
   `main` ואת התיקייה `/docs`, ושמור. זו תהיה כתובת הדשבורד.
6. לשונית Actions → בחר את "site-monitor" → לחץ "Run workflow"
   כדי להריץ בפעם הראשונה (זה ייצור את צילום הבסיס).

## שינוי תדירות הבדיקה
בקובץ `.github/workflows/monitor.yml`, השורה:
`- cron: '0 */6 * * *'` פירושה "כל 6 שעות". אפשר לשנות
(למשל `'0 */3 * * *'` = כל 3 שעות).

## הוספת עמודים לניטור
בקובץ `monitor.js`, בתוך `SITES`, הוסף שורות בפורמט:
`{ name: 'שם-יחודי', url: 'https://...' },`
